## 你的任务

快速确认标注是否大致正确。**你不需要给出修正坐标**，只需要判断哪些元素有明显问题。

## 判断标准

### 通过（可接受）
- 标注框大致覆盖了该物件的视觉主体
- 框包含了物件的屋顶/遮阳棚/招牌 → 通过（这些都是物件的一部分）
- 偏小一些完全可以接受
- 边缘轻微偏移均可接受
- 框稍微压到旁边的道路/地面边缘 → 通过
- 不要求像素级精确

### 不通过（明显有问题）
- 标注框**完全偏离**了目标物件（比如只有一小部分标在了目标物件上，和预期位置差距很大）
- 元素 id 与实际标注位置的物件明显不符（如标注框框住的是一口井，但 id 是"糕点摊"）

### 重要原则
- 只要标注框和目标物件**有明显重叠**，就应该判为通过
- 不要因为框的大小不够精确而否决

## 返回格式

请返回 JSON（不要包含其他内容）：

如果所有元素均可接受：
```json
{ "pass": true, "problematic_element_ids": [], "feedback": {} }
```

如果有明显问题的元素：
```json
{
  "pass": false,
  "problematic_element_ids": ["有问题的元素id1", "有问题的元素id2"],
  "feedback": {
    "有问题的元素id1": "简述该元素标注错误的原因，如：标注位置完全偏离，实际应在道路右侧",
    "有问题的元素id2": "简述原因"
  }
}
```

## 重要
- 只标记**明显错误**的元素，不要标记"不够完美但基本正确"的元素
- 被标记的元素会被重新标注，所以请**非常谨慎**地判断
- `feedback` 中对每个被标记的元素给出**简短具体**的错误原因（一句话即可），用于指导下次标注
- 如果不确定某个元素是否有问题，**请判为通过**
- 如果所有元素都大致正确，请返回 pass: true
