## 最高优先级：固定导航模板
输入图不是最终美术，而是空间布局契约：
- 模板中的浅色连续区域 = 最终地图里必须清晰可见、宽阔连续的可行走地面/地毯/平台。
- 模板中的深色区域 = 最终地图里的墙、家具、软装、阴影边界或不可走背景。
- 最终地图的可行走地面轮廓必须保留模板的 **mask silhouette**：外轮廓、左右凹口、中心暗色阻挡岛、暗色边界都必须保持在同一位置。允许像素美术自然修边，但不能改变拓扑。
- 不允许新增模板里没有的可走走廊、侧厅、房间入口或地面口袋；不允许把模板暗色区域画成明亮地面；不允许把模板浅色区域画成床、柜台、沙发、墙、厚阴影或不可走家具。
- 不要把模板画成 UI、调试图、图例、网格、框选层或色块叠加层；请把它转译成自然的像素空间。

## 必须遵守的硬性约束
1. 输出必须是 9:16 竖屏、接近 90 度俯视的 2D 像素游戏地图，适合规范化为 900×1600 Phaser 背景图。
2. 可走区以模板为准：出生点、中心活动区、每个互动点停靠区必须落在模板浅色地面上，并且从出生点到每个互动点都有一条宽、简单、连续的视觉地面路径。
3. 核心通道视觉宽度至少约 140px；不要用碎石、花丛、杂乱阴影、灯光斑点、地毯裂缝或碎装饰把主路切成小洞、小岛、锯齿或迷宫。
4. 地图里的大型家具、墙体、装饰、床、柜台、控制台、沙发、货架等必须待在模板深色区域或浅色区域边缘，不能遮盖浅色主地面，也不能挡住热点停靠区。
5. 可交互 Agent 和重点道具会由独立 sprite 叠加。地图中只能为它们留下自然空地、家具旁留白、地毯边缘或灯光下停靠点；不要直接画出 Agent/NPC 身体，也不要画出需要叠加的重点道具本体。
6. 热点位置只能通过自然环境组织表达，不允许画任何可见定位标记。
7. 绝对不要生成主角、访客、互动 Agent/NPC 的身体、角色立绘、道具立绘、多角色拼贴或宠物。
8. 绝对不要生成任何文字、标签、招牌文字、编号、图例、Logo、水印、坐标、方向说明或帧号。
9. 绝对不要生成 dashed boxes、dashed circles、selection outlines、target reticles、UI markers、coordinate dots、placeholder frames、bounding boxes、十字准星、虚线框、虚线圆、坐标点、编辑器选区感图形、网格线或审查标记。
10. 视觉可以有主题家具和情绪细节，但地面边界必须清楚；用户打开可走区 overlay 时，应能看出 overlay 和地图地面基本重合。
11. 不要做旧 WorldX 横屏大地图、开放世界、IP 复刻、知名品牌复刻或受版权保护角色/建筑复刻。
12. 如果模板和文字描述冲突，以模板的可走区形状为准；文字描述只决定材质、气质和软装风格。
13. 生成完成后会用另一套视觉模型重新标注“它认为图中真实可走的区域”，并与模板做 IoU 对比；如果你没有保留模板 mask，会被判失败重试。
