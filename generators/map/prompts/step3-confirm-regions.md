## 你的任务

快速确认标注是否大致正确。**你不需要给出修正坐标**，只需要判断哪些区域有明显问题。

## 判断标准

### 通过（可接受）
- 标注框大致覆盖了该区域的核心范围
- 偏小一些完全可以接受
- 边缘轻微偏移、贴边、不完美均可接受
- 不要求像素级精确

### 不通过（明显有问题）
- 标注框**完全偏离**了目标区域（标在了错误的位置）
- 标注框**严重偏大**，将大量墙外/路面/其他区域的空间框了进去
- 标注框覆盖了**另一个功能区**的主体范围
- 区域 id 与实际位置明显不符（如 id 含 "northwest" 但标注在东南）

### 容忍范围
- 框稍微压到墙体边缘 → 通过
- 框比理想范围偏小 20-30% → 通过
- 室外区域框到了少量边缘道路 → 通过

## 返回格式

请返回 JSON（不要包含其他内容）：

如果所有区域均可接受：
```json
{ "pass": true, "problematic_region_ids": [], "feedback": {} }
```

如果有明显问题的区域：
```json
{
  "pass": false,
  "problematic_region_ids": ["有问题的区域id1", "有问题的区域id2"],
  "feedback": {
    "有问题的区域id1": "简述该区域标注错误的原因，如：标注位置完全偏离，实际应在地图左下角",
    "有问题的区域id2": "简述原因"
  }
}
```

## 重要
- 只标记**明显错误**的区域，不要标记"不够完美但基本正确"的区域
- 被标记的区域会被重新标注，所以请谨慎判断
- `feedback` 中对每个被标记的区域给出**简短具体**的错误原因（一句话即可），用于指导下次标注
- 如果所有区域都大致正确，请返回 pass: true
