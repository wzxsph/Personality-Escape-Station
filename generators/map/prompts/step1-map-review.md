## 约束条件
1. 视觉风格是否适合「人格出逃空间站 / Personality Escape Station」：竖屏像素人格空间、黑色幽默但温柔、边界清晰、适合作为 Phaser 背景图。
2. 是否接近俯视视角？（不是透视、不是等距，是80度以上俯视或接近俯视的视角）
3. 是否遵守了地图规划？尤其是 `buildingMode`、整体场景功能，以及在需要时才出现的功能区。
4. 下方“功能区”中的被规划为**可进入建筑**的建筑，必须只有一层且是被横切的状态（没有屋顶），以展示内部；景观建筑允许保留完整屋顶与外立面。
5. 若整个场景在一个建筑/房间内，且根据场景描述判断为封闭建筑（如末日避难所、宇宙飞船），则该建筑不需要任何入口；若整个场景在一个建筑/房间内且不是封闭建筑，则有无入口皆可。该建筑也必须是没有屋顶、能直接看见内部的状态。
6. 若整个场景不是在一个建筑/房间内，则所有可进入建筑可进入建筑都必须没有门，入口是墙壁上的缺口，缺口处是开放的"| |"型，而不是"n"型（缺口上方不能有墙体、牌匾等任何东西）。
7. 景观建筑虽然可保留完整外观，但是否过高/过大，以至于显著遮挡主要道路？
8. 可进入建筑内部是否有合理的家具/物品？地面是否留有足够的行走空间？
9. 地图中是否出现了人物/角色？（不应该有）
10. 不能有任何说明文字、标签、招牌文字、图例或符号化 UI。
11. 各区域之间是否有清晰的道路/路径相连？
12. 路面是否有被高大物体（如树木）显著遮挡的情况？
13. 整体布局是否合理美观？
14. 建筑边界、道路、室内外分界和主要功能区是否足够清晰，便于后续做区域定位与可行走区域识别？
15. "可交互元素"中列出的元素是否在地图中自然出现？（不要求严格位置，只要大致存在且视觉上可辨认即可）
16. 是否避免了 IP 复刻、旧 WorldX 横屏大地图、角色人物、品牌标识和版权化建筑。
17. 是否是 9:16 竖屏构图，核心区域、出生点、互动 Agent 和道具是否落在安全区。
18. 从预期出生点到每个可交互元素是否有连续、清晰、可标注的可行走路径。
19. 地图中是否完全没有可见定位/调试标记？任何 dashed boxes、dashed circles、selection outlines、target reticles、coordinate dots、placeholder frames、bounding boxes、虚线框、虚线圆、十字准星、坐标点、图例、编辑器选区感图形都必须判失败。
20. 主可走路径是否足够简单、宽、连续？如果核心通道被碎石、花丛、阴影、灯光斑点或细碎装饰切成复杂小洞/孤岛/锯齿裂缝，必须判失败。
21. 互动点是否以自然环境留白呈现？如果互动点被画成白色/彩色边框、圆圈、方框、光标或地面 UI，必须判失败。
